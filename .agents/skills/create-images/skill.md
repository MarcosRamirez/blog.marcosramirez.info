## Skill: Image Generation

### ⚠️ MANDATORY PROCESS BEFORE GENERATING

**NEVER generate an image without following these steps:**

1. **READ the complete post** before generating the image
2. **IDENTIFY in the post:**
   - The main concept / central theme of the post (e.g.: piracy, automation, home automation, finance)
   - Brand names (Netflix, Google, Amazon, etc.)
   - Service names (Jellyfin, Home Assistant, AdGuard, etc.)
   - Company or product names mentioned
3. **INCLUDE in the prompt (BOTH are mandatory):**
   - **The post's main concept/theme** (maximum priority)
   - The mentioned brands/services
   - The appropriate visual tone

### Example of correct flow:

```
1. Read post about "Why people turn back to piracy"
2. Identify:
   - Main concept: PIRACY, P2P, torrent downloading
   - Brands: Netflix, Disney+, Amazon Prime, Apple TV
3. Generate prompt: "piracy, torrent downloading on laptop, P2P file sharing, streaming apps on multiple screens, dark realistic photo, no text, no watermarks"
```

### Using the Local Script (`_tools/image-generator/image_generator.js`)
The script `_tools/image-generator/image_generator.js` generates images trying first **Nano Banana (Google Vertex AI)**. If it fails, it will use fallback providers.

### 🌟 Golden Rule: Priority to Nano Banana
The blog's goal is that **all** images are generated by Nano Banana.
To know which provider generated the image, the script **automatically modifies the final filename**:
- If Nano Banana works: `your-file-nanobanana.png`
- If fallback enters: `your-file-pollinationsai.png` or `-huggingface.png`

**Mandatory instruction after generating an image:**
You must read the `finalPath` property from the JSON returned by the script and **update the post's frontmatter (`image:`)** with that exact path.
If you see old posts with images that don't have the `-nanobanana suffix, you must regenerate them with this script to migrate them gradually.

### How to use it

```bash
node _tools/image-generator/image_generator.js \
  --prompt "<visual description in English>" \
  --output "<path/where/to save/base-name.png>"
```

*Note: Pass the base name (e.g. `post.png`). The script takes care of adding the `-nanobanana` or `-pollinationsai` suffix before saving.*

The `--width` and `--height` parameters are optional. By default it generates panoramics at **1900x478px**.

### Good practices for the prompt
- **NEVER generate a prompt without first reading the post**
- **Images MUST be photorealistic/realistic, NOT drawing, illustration or cartoon style**
- **DO NOT use the word "icon" for logos** - logos don't look real
- **To represent brands/services**, use visual concepts instead:
  - Instead of "Netflix icon" → "streaming on laptop screen"
  - Instead of "AdGuard icon" → "DNS settings screen"
  - Instead of "Home Assistant logo" → "smart home dashboard on tablet"
- Always write it in English.
- Be concrete and visual: describe scene, style, colors, lighting.
- Always include at the end: `no text, no watermarks`.
- **Examples of correct prompts:**
  - Post about Netflix: "Netflix streaming app, dark theme, movie night, no text, no watermarks"
  - Post about Home Assistant: "Home Assistant smart home dashboard, IoT devices, dark modern UI, no text, no watermarks"
  - Post about AdGuard: "AdGuard DNS blocker, privacy shield, ad blocking interface, no text, no watermarks"

### Interpreting the response
The script always returns a JSON line in stdout:

```json
{ "status": "SUCCESS", "code": 0, "message": "Image saved at: ...", "provider": "Pollinations.ai" }
{ "status": "ERROR", "code": 2, "message": "All providers failed: ..." }
```

### ⚠️ Error Management — MANDATORY

**Code `0` — Success:**
The image is available at the path indicated in `--output`. Continue.

**Code `1` — Incorrect parameters:**
Fix the parameters and retry.

**Code `2` — All providers exhausted:**
- **STOP immediately. Do not retry. Do not continue with other tasks.**
- Inform the user with this message: *"The image generation service is not available right now. All free providers have been exhausted. Please try again later."*
- Wait for instructions from the user before doing anything else.

**Code `4` — Unexpected error:**
- **STOP immediately.**
- Inform the user of the exact error message returned by the script.
- Wait for instructions before continuing.
